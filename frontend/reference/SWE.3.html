SWE.3 Software Detailed Design Specification

1. Introduction

Approved

This Software Detailed Design (SDD) document specifies the internal structure and behavior of the Brake Controller Software Unit. It serves as the primary technical reference for implementation and verification, ensuring compliance with ASPICE SWE.3 requirements.

The scope includes the definition of software unit interfaces, internal data structures, and the dynamic behavior required to meet the functional safety goals defined in the Software Architecture (SWE.2).

2. Unit Design - Brake Controller

Approved

The Brake Controller unit implements the core logic for Electronic Braking Systems (EBS). It processes wheel speed sensor data and pressure requests to modulate hydraulic valves via high-frequency PWM signals.

Figure 2.1: Controller Logic Flow

Key sub-modules include the Pressure Estimator, Slip Control Logic, and the Valve Driver Interface. Each module is designed for deterministic execution within a 5ms task cycle.

3. Interfaces

Approved

The unit exposes standardized AUTOSAR Sender-Receiver and Client-Server interfaces for inter-unit communication.

In Review

The dynamic behavior is modeled as a hierarchical state machine. The system transitions from 'Standby' to 'Active' upon detection of a valid brake request signal.
